建议优先部署到 **Cloudflare Pages**，比 GitHub Pages 更适合这个 reader：支持自定义构建命令、环境变量、预览部署，配置也更直接。Cloudflare 官方 Vite Pages 指南也是用 build command + output directory 的方式部署。参考：[Cloudflare Pages Vite guide](https://developers.cloudflare.com/pages/framework-guides/deploy-a-vite3-project/)。

**部署前先确认**

本项目 reader 是静态 Vite app：

```bash
pnpm reader:build
```

输出目录：

```text
dist/reader
```

需要的公开环境变量：

```bash
VITE_OSS_BASE_URL=https://guanmo-obsidian.oss-cn-shanghai.aliyuncs.com
VITE_OSS_INDEX_PATH=public-index.json
```

这些不是密钥，可以放到 Cloudflare/GitHub 的构建环境变量里。

---

**方案 A：Cloudflare Pages 推荐**

1. 把代码推到 GitHub。

2. 进入 Cloudflare Dashboard：

```text
Workers & Pages -> Create application -> Pages -> Import an existing Git repository
```

3. 选择你的 GitHub 仓库。

4. 构建配置：

```text
Framework preset: None / Vite 都可以
Build command: pnpm reader:build
Build output directory: dist/reader
Root directory: /
Node version: 20 或 22
```

Cloudflare 文档里 Vite 示例也是设置 build command 和 output directory。参考：[Cloudflare Pages Vite guide](https://developers.cloudflare.com/pages/framework-guides/deploy-a-vite3-project/)。

5. 添加环境变量：

```text
VITE_OSS_BASE_URL=https://guanmo-obsidian.oss-cn-shanghai.aliyuncs.com
VITE_OSS_INDEX_PATH=public-index.json
```

6. 处理 SPA 路由刷新。

需要加一个文件：

```text
apps/reader/public/_redirects
```

内容：

```text
/* /index.html 200
```

否则 `/articles/xxx` 直接刷新可能 404。

7. OSS CORS 里加入 Cloudflare Pages 域名：

```text
https://你的项目.pages.dev
https://你的正式域名
```

保留本地开发域名：

```text
http://localhost:5173
http://127.0.0.1:5173
```

**方案 B：GitHub Pages**

GitHub Pages 可以用 GitHub Actions 自定义 workflow 部署。GitHub 官方文档说明自定义 workflow 需要使用 `configure-pages`、`upload-pages-artifact`、`deploy-pages` 等 action。参考：[GitHub Pages custom workflows](https://docs.github.com/en/pages/getting-started-with-github-pages/using-custom-workflows-with-github-pages)。

新增文件：

```text
.github/workflows/deploy-reader.yml
```

内容示例：

```yaml
name: Deploy Reader

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v6
        with:
          node-version: 22
          cache: pnpm

      - run: pnpm install --frozen-lockfile

      - run: pnpm reader:build
        env:
          VITE_OSS_BASE_URL: https://guanmo-obsidian.oss-cn-shanghai.aliyuncs.com
          VITE_OSS_INDEX_PATH: public-index.json

      - name: SPA fallback
        run: cp dist/reader/index.html dist/reader/404.html

      - uses: actions/configure-pages@v5

      - uses: actions/upload-pages-artifact@v4
        with:
          path: dist/reader

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

然后到 GitHub 仓库：

```text
Settings -> Pages -> Build and deployment -> Source: GitHub Actions
```

注意：如果 GitHub Pages 地址是：

```text
https://你的用户名.github.io/仓库名/
```

Vite 可能需要配置 `base`，否则静态资源路径可能不对。Cloudflare Pages 通常不需要这个问题，所以我更建议先用 Cloudflare。

**我的建议**

先用 Cloudflare Pages：

```text
Build command: pnpm reader:build
Output: dist/reader
Env: VITE_OSS_BASE_URL / VITE_OSS_INDEX_PATH
```

然后给 OSS CORS 加上 Cloudflare 域名即可。
